# Library Directory

## Structure

- `supabase/` - Supabase client configurations (client, server, middleware)
- `google/` - Google API integrations (Calendar, Tasks, OAuth)
- `scheduling/` - Auto-scheduling algorithms and logic
- `workflows/` - Workflow state transition logic
- `utils.ts` - General utility functions

## Guidelines

- Keep business logic separate from UI components
- Write unit tests for all utility functions
- Document complex algorithms
