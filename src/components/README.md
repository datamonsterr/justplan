# Components Directory

## Structure

- `ui/` - Reusable UI components (buttons, forms, modals) using Shadcn UI
- `calendar/` - Calendar-related components
- `tasks/` - Task management components
- `workflows/` - Workflow state management components

## Conventions

- Use TypeScript for all components
- Follow compound component pattern where appropriate
- Co-locate tests with components (`ComponentName.test.tsx`)
- Use Tailwind CSS for styling
